# 2. 선형대수학

## 개요

3D 그래픽스는 점·벡터·행렬의 연산이 전부다.
화면에 픽셀을 찍기까지 거치는 **월드 → 뷰 → 프로젝션 → 스크린** 변환과,
**라이팅·컬링·물리** 판정 대부분이 내적/외적 한두 개로 환원된다.
"수학"이 아니라 "도구 사용법"으로 익혀야 한다.

## 좌표계 함정

3D 엔진마다 좌표 규약이 다르다.

| 엔진/API | Up 축 | Forward 축 | 행/열 우선 |
| --- | --- | --- | --- |
| **Unreal Engine** | +Z | +X | 열 우선 (수학적), 메모리는 행 우선 |
| **Unity** | +Y | +Z | 열 우선 |
| **DirectX** | +Y | +Z | 행 우선 |
| **OpenGL** | +Y | -Z | 열 우선 |

좌표계 변환을 잘못하면 모델이 거꾸로 서거나 좌우가 바뀐다.
**다른 엔진 간 자산 임포트 시 가장 흔한 버그.**

## 동차 좌표 (Homogeneous Coordinates)

- 3D 점을 (x, y, z, 1), 벡터를 (x, y, z, 0)으로 4D 표현
- 4x4 행렬로 평행이동까지 한 번에 표현 가능
- w 컴포넌트의 차이가 점/벡터의 변환 차이를 만든다 (벡터는 평행이동 무시)

## 학습 순서 추천

1. **내적/외적** — 가장 자주 쓰임, 직관 익히기 쉬움
2. **행렬 변환** — 이걸 모르면 셰이더를 못 읽음
3. **시야각/컬링** — 위 둘의 응용

## 심화 학습 키워드

- Dual Quaternion (스키닝)
- Plücker 좌표 (직선 표현, 충돌)
